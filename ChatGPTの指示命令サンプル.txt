# 開発概要
- エクセルファイルを読みこみそれを解析したデータを別エクセルファイルに転写する簡易ツール

# 開発環境表
| 項目                | 希望                                                  | メモ                           |
| ----------------- | ------------------------------------------------------ | ---------------------------- |
| **OS**            | Windows 11                                         | WSL2 や Docker を使わない      |
| **Python のバージョン** | 3.12                                             | インストールしていない    |
| **仮想環境管理**        | venv                            | 非エンジニアでも構築できるようにしたい        |
| **エディタ / IDE**    | VS Code + Neovim プラグイン                             | VSCode内でデバッグやテストできる用に.vcodeフォルダのlauch.json/setting.jsonファイルが必要             |
| **エディタのデバッグ**    | python デバッガ（標準）                             | 標準のみで対応             |
| **パッケージ管理**       | pip                                     | 依存パッケージはrequirements.txtのみ |
| **コード整形 / 静的解析**  | black, ruff, mypy など                                   | CI 連携の無 / 将来的にも入れない                    |
| **テストフレームワーク**    | pytest                                               | カバレッジ計測や自動化の否               |
| **実行形態**          | GUI                  | エクセルを読み込み、エクセルを出力するツール               |
| **配布方法**          | exe / app | バッチ/シェルで作成できるようにする               |
| **macOS サポート**          | python環境が初め存在しないのでシェルでpython環境を入れる | Homebrewもなければシェルで入れる|
| **CI/CD**         | 未使用                    |               |
|  **ツール実行環境**        | win/mac                    |  ユーザーがwin/mac両方使うから             |
|  **ライブラリ**        | GUIはtkinter, エクセルはopenpyxlのみで良い                      |               |

# 開発内容
| 機能一覧                | 希望                                                  | メモ                           |
| ----------------- | ------------------------------------------------------ | ---------------------------- |
| **エクセルファイルの読み書き**            | エクセルファイルを読み込み・書き込みを行う                                         |   |
| **GUIでエクセルファイルの出力先をフォルダ指定**            | ボタンをクリックするとフォルダ指定のダイアログを出してユーザーが指定できる.指定したフォルダのパスをGUI上で表示. | 指定したフォルダパスを保存して再指定コストを減らす  |
| **GUIでデータフォルダ指定**            |    ボタンをクリックするとフォルダ指定のダイアログを出してユーザーが指定できる.指定したフォルダのパスをGUI上で表示.    | 指定したフォルダパスを保存して再指定コストを減らす  |
| **GUIで処理進捗バーを表示**            |    0から100%を進捗バーで表示                                      |   |
| **GUIで処理のログ一覧を表示**            |   prinfなどのログを出力                                       |   |
| **ログを保存**            |   ログをファイルで出力                                       |  ログファイルを収めるフォルダがなければ作成.作成するフォルダ位置はツール直下 |
| **設定内容を保存**            |   エクセルファイルの出力先/データフォルダ指定のフォルダパスを保存して再度GUIを開いた時は保存したのを反映                                       | 設定ファイルを収めるフォルダがなければ作成.作成するフォルダ位置はツール直下  |
